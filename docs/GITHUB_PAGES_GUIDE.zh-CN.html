Setup Guide

GitHub Pages 个人博客部署指南

静态博客 · 中英文版本 · 无需构建工具

1. 创建仓库

个人主页推荐仓库名为 你的 GitHub 用户名.github.io。例如用户名是 georgezhong10-cmyk，仓库名就是 georgezhong10-cmyk.github.io。

2. 初始化并推送代码

git init
git add .
git commit -m "Create bilingual GitHub Pages blog"
git branch -M main
git remote add origin https://github.com/你的用户名/你的用户名.github.io.git
git push -u origin main

3. 开启 GitHub Pages

  1. 进入 GitHub 仓库页面，打开 Settings。
  2. 点击左侧 Pages。
  3. Build and deployment 选择 Deploy from a branch。
  4. Branch 选择 main，目录选择 /root。
  5. 点击 Save，等待 GitHub 发布完成。

4. 替换个人信息

优先修改 index.html、en/index.html、两篇示例文章、_config.yml、robots.txt 和 sitemap.xml。把标题、简介、GitHub 用户名、站点地址和文章内容换成你的真实信息。

5. 写新文章

复制 posts/zh/hello-github-pages.html 或 posts/en/hello-github-pages.html，改文件名、标题、日期、正文和语言切换链接，然后在首页与归档页增加入口。

6. 自定义域名，可选

如果你有自己的域名，在根目录新建 CNAME 文件，内容只写域名；然后在 DNS 中添加 CNAME 记录指向 你的用户名.github.io，最后到 Settings → Pages 填写 custom domain 并启用 HTTPS。

7. 本地预览，可选

python3 -m http.server 8000

然后访问 http://localhost:8000。

提示：如果发布后样式没加载，通常是路径或 Pages 分支设置问题。个人主页仓库使用 用户名.github.io 会让路径最简单。
