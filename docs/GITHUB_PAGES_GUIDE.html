GitHub Pages Personal Blog Setup Guide

Static blog · Bilingual pages · No build tool required

1. Create the repository

For a personal site, name the repository your-github-username.github.io. For example, if your username is georgezhong10-cmyk, use georgezhong10-cmyk.github.io.

2. Initialize and push the code

git init
git add .
git commit -m "Create bilingual GitHub Pages blog"
git branch -M main
git remote add origin https://github.com/your-github-username/your-github-username.github.io.git
git push -u origin main

3. Enable GitHub Pages

  1. Open the repository on GitHub and go to Settings.
  2. Open Pages from the sidebar.
  3. Under Build and deployment, choose Deploy from a branch.
  4. Select main as the branch and /root as the folder.
  5. Save and wait for GitHub to publish the site.

4. Replace personal details

Start with index.html, en/index.html, the two sample posts, _config.yml, robots.txt, and sitemap.xml. Replace the title, bio, GitHub handle, production URL, and post content.

5. Write a new post

Copy posts/zh/hello-github-pages.html or posts/en/hello-github-pages.html, rename it, update the title, date, body, and language-switch link, then add the post to the home page and archive.

6. Optional custom domain

If you own a domain, create a root-level CNAME file containing only that domain. Add a DNS CNAME record pointing to your-github-username.github.io, then configure the custom domain in Settings → Pages and enable HTTPS.

7. Optional local preview

python3 -m http.server 8000

Then open http://localhost:8000.

Tip: If styles do not load after publishing, check asset paths and Pages branch settings. A username.github.io repository keeps paths simplest.
