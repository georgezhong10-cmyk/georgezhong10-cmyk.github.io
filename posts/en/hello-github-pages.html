Build a small technical blog with GitHub Pages

May 10, 2026 · Blogging · Static Site

A personal technical blog does not need to start with a heavy framework. For early writing, the important pieces are clear structure, stable publishing, and low maintenance cost. GitHub Pages is a good fit: push HTML, CSS, images, and posts to a repository, then enable Pages from the repository settings.

What this blog includes

This project contains a Chinese home page, an English home page, an archive, bilingual sample posts, shared styles, a small navigation script, and deployment guides. You can replace the biography, GitHub handle, email, post titles, and post content with your own details.

Recommended structure

.
├── index.html
├── en/
├── archive/
├── posts/
├── assets/
├── docs/
├── _config.yml
└── .nojekyll

Publishing steps

  1. Create a GitHub repository. For a personal site, use your-username.github.io.
  2. Push every file in this project to the repository's main branch.
  3. Open Settings → Pages in the repository.
  4. Under Build and deployment, choose Deploy from a branch.
  5. Select main as the branch and /root as the folder, then save.
  6. Wait for GitHub to publish the site, then visit https://your-username.github.io.

Writing the next post

Copy this post file, update the title, date, body, and language-switch link, then add the new entry to the home page and archive. If the blog grows, you can later migrate to Jekyll, Astro, or Hugo. Until then, keeping the site simple makes writing easier.

A small rule: write three to five useful posts before adding comments, advanced taxonomy, or a build pipeline. The scarce resource is not tooling. It is clear, durable writing.
