用 GitHub Pages 搭一个轻量个人技术博客

2026-05-10 · Blogging · Static Site

一个个人博客不一定要从复杂框架开始。对技术写作来说，最重要的是结构清晰、发布稳定、 内容易维护。GitHub Pages 正好适合这个阶段：把 HTML、CSS、图片和文章推到仓库， 开启 Pages 后就可以访问。

这套博客包含什么

当前项目已经包含中文首页、英文首页、文章归档、中英文示例文章、样式文件、交互脚本和 GitHub Pages 部署指南。你可以直接替换个人介绍、GitHub 用户名、邮箱、文章标题和内容。

推荐仓库结构

.
├── index.html
├── en/
├── archive/
├── posts/
├── assets/
├── docs/
├── _config.yml
└── .nojekyll

发布步骤

  1. 在 GitHub 新建仓库，个人博客推荐命名为 你的用户名.github.io。
  2. 把本项目所有文件推送到仓库的 main 分支。
  3. 进入仓库的 Settings → Pages。
  4. 在 Build and deployment 中选择 Deploy from a branch。
  5. Branch 选择 main，目录选择 /root，保存。
  6. 等待 GitHub 完成发布，访问 https://你的用户名.github.io。

后续怎么写新文章

复制这篇文章文件，改标题、日期、正文和语言切换链接；然后在首页和归档页加上新文章入口。 如果文章变多，可以进一步迁移到 Jekyll、Astro 或 Hugo，但前期保持简单会更有利于持续写作。

小建议：先写 3 到 5 篇高质量笔记，再考虑复杂分类、评论系统或自动构建。 博客最稀缺的不是框架，而是持续可读的内容。
